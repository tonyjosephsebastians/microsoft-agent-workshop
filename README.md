# Microsoft Agent Workshop

Small Python workshop project for calling Azure OpenAI through Microsoft Agent Framework.

## Requirements

- Python 3.12+
- `uv`
- An Azure OpenAI resource and deployment

## Setup

Install dependencies:

```powershell
uv sync
```

Create a `.env` file in the project root:

```env
AZURE_OPENAI_ENDPOINT=https://YOUR-RESOURCE.openai.azure.com/
AZURE_OPENAI_API_KEY=YOUR_API_KEY
AZURE_OPENAI_MODEL=YOUR_DEPLOYMENT_NAME
AZURE_OPENAI_API_VERSION=preview
```

The script also accepts these model env var aliases:

- `AZURE_OPENAI_CHAT_MODEL`
- `AZURE_OPENAI_MODEL_NAME`
- `AZURE_OPENAI_DEPLOYMENT`

If `AZURE_OPENAI_API_KEY` is omitted, the script falls back to `AzureCliCredential`, so you can authenticate with:

```powershell
az login
```

## Run

```powershell
uv run python .\main.py
```

Expected behavior: the script creates a simple `HaikuBot` agent and prints a haiku response generated by your Azure OpenAI deployment.

## Files

- `main.py`: sample agent entrypoint
- `pyproject.toml`: project metadata and dependencies
- `uv.lock`: locked dependency graph

## Notes

- `.env` is ignored by git and is not committed.
- The current script normalizes old dated Azure API versions such as `2024-12-01-preview` to `preview` for the v1 Responses flow.
- This sample uses the current `OpenAIChatClient` pattern instead of the older `AzureOpenAIResponsesClient` import.
